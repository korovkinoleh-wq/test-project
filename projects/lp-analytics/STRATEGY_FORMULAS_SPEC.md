# Формальная спецификация формул стратегии LP

Источник: AI Studio export от 2026-03-27 (`projects/lp-analytics/inbox/aistudio-drop-2026-03-27/STRATEGY_FORMULAS_SPEC.md`).

Этот документ описывает математическую логику, используемую в модулях **Strategy** и **Rebalance & Liquidation Risk**.

---

## 1. Обозначения и единицы измерения

| Символ | Описание | Единицы |
| :--- | :--- | :--- |
| $C$ | Общий капитал (Total Capital) | USD (USDC) |
| $P$ | Текущая рыночная цена актива | USD / Asset |
| $W$ | Ширина диапазона (Range Width) | USD (Price Units) |
| $A_{pct}$ | Доля актива в диапазоне (Asset %) | % (0-100) |
| $U_{pct}$ | Доля USDC в диапазоне (USDC %) | % (0-100) |
| $APR_{ref}$ | Доходность эталонного пула | % годовых |
| $k$ | Мультипликатор доходности (Yield Multiplier) | Безразмерный |
| $L$ | Нижняя граница (Lower Bound) | USD |
| $H$ | Верхняя граница (Upper Bound) | USD |
| $Q_{asset}$ | Количество единиц актива | Asset Units |

---

## 2. Базовая калибровка (Reference Calibration)

Система масштабирует данные эталонного пула (Reference Pool) под текущую цену.

### 2.1. Геометрическая цена эталона
Для Uniswap V3 ликвидность распределена относительно геометрического среднего:
$$P_{ref} = \sqrt{L_{ref} \times H_{ref}}$$

### 2.2. Коэффициент масштабирования (Price Scale)
Позволяет пересчитать ширину диапазона эталона под текущую цену:
$$S_{price} = \frac{P}{P_{ref}}$$

---

## 3. Модуль Strategy (Accumulation / Distribution)

### 3.1. Расчет мультипликатора $k$
**В режиме Auto Split (от целевого дохода $I_{target}$):**
$$I_{ref} = \frac{C \times (APR_{ref} / 100)}{12}$$
$$k = \frac{I_{target} \times \text{Share}_{asset}}{I_{ref}}$$

**В режиме Manual Range (от заданной ширины $W$):**
$$k = \frac{W_{ref} \times S_{price}}{W}$$

### 3.2. Асимметричные границы
Диапазон распределяется вокруг текущей цены $P$ согласно пропорциям $A_{pct}$ и $U_{pct}$:
1.  **Отступ вниз (Downside):** $\Delta_{down} = W \times \frac{U_{pct}}{100}$
2.  **Отступ вверх (Upside):** $\Delta_{up} = W \times \frac{A_{pct}}{100}$
3.  **Границы:**
    $$L = P - \Delta_{down}$$
    $$H = P + \Delta_{up}$$

### 3.3. Анализ при достижении нижней границы (Lower Bound Analysis)
Расчет количества актива, который будет на руках, если цена упадет до $L$:
1.  **Начальные активы:** $Q_0 = \frac{C \times (A_{pct}/100)}{P}$
2.  **Купленные активы (из USDC):** $Q_{buy} = \frac{C \times (U_{pct}/100)}{\sqrt{L} \times \sqrt{P}}$
3.  **Итого:** $Q_{total} = Q_0 + Q_{buy}$
4.  **PnL:** $\text{PnL}_L = (Q_{total} \times L) - C$

## 4. Модуль Rebalance & Liquidation Risk

Моделирует использование $Q_{total}$ в качестве залога для займа USDC и открытия новой позиции.

### 4.1. Кредитное плечо и Долг
1.  **Стоимость залога:** $V_{coll} = Q_{total} \times P_{reb}$
2.  **Рабочий капитал:** $C_{new} = V_{coll} \times \frac{LTV}{100}$
3.  **Сумма займа:** $D_0 = C_{new} \times \frac{U_{pct\_new}}{100}$
4.  **Итоговый долг:** $D_{total} = D_0 \times (1 + \frac{Rate}{100} \times \frac{Days}{365})$

### 4.2. Риски ликвидации
1.  **Health Factor (HF):**
    $$HF = \frac{V_{coll} \times (LT / 100)}{D_{total}}$$
2.  **Цена ликвидации (Liq Price):**
    $$P_{liq} = \frac{D_{total}}{Q_{total} \times (LT / 100)}$$

### 4.3. Эффект «Доливки» (Top-up Effect)
Новая LP-позиция содержит актив, который увеличивает общий залог:
1.  **Новый актив в залог:** $Q_{new} = \frac{C_{new} \times (A_{pct\_new}/100)}{P_{reb}}$
2.  **Финальная цена ликвидации:**
    $$P_{liq\_final} = \frac{D_{total}}{(Q_{total} + Q_{new}) \times (LT / 100)}$$

---

## 5. Константы и допущения
*   **Месяц:** Принимается равным $365.25 / 12$ дней.
*   **Uniswap V3 Math:** Используется упрощенная модель ликвидности (плоская плотность внутри диапазона), что дает погрешность <1% при малых диапазонах.
*   **Цены:** Обновляются каждые 30-60 секунд из Binance API.
