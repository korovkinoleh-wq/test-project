# LP Analytics

Проект по аналитике LP (Liquidity Provider) в DeFi.

## Цель

Построить продукт, который на основе реальных пулов, snapshots и расчётной модели помогает оценивать комиссии/доходность в заданном диапазоне ликвидности и поддерживает стратегический слой поверх калькулятора.

## Принципы

- Истина проекта хранится в файлах, а не в истории чата.
- Любая важная договорённость фиксируется письменно.
- Любая новая сессия должна быстро входить в проект через `SESSION_START.md`.
- Формулы, допущения и ограничения должны быть явными.
- Любая задача проходит через понятный workflow, а не через импровизацию.

## Главные входы

Сначала читать:
1. `SESSION_START.md`
2. `STATUS.md`
3. `BACKLOG.md`
4. `DECISIONS.md`
5. профильный документ по задаче (`FORMULAS.md`, `DATA_MODEL.md`, `ARCHITECTURE.md`)
