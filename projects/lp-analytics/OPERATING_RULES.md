# OPERATING_RULES

## Project memory rules

- После значимого этапа обновлять `STATUS.md` и `WORKLOG.md`.
- После важного решения обновлять `DECISIONS.md`.
- После изменения математики обновлять `FORMULAS.md`.
- После изменения данных/схемы обновлять `DATA_MODEL.md`.
- После изменения общей структуры обновлять `ARCHITECTURE.md`.

## Communication rules

- Пользователь видит итоговую координацию через одного интерфейсного агента.
- Межагентные взаимодействия доступны пользователю в виде summary по умолчанию и full audit по запросу.

## Delivery rules

- Не закрывать задачу без фиксации нового проектного состояния.
- Не держать критические допущения только в голове/контексте.
- Любой воспроизводимый шаг должен быть записан либо в коде, либо в документации.
