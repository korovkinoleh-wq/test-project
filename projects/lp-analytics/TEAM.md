# TEAM

## Main operating model

Пользователь общается только с координатором. Остальные роли — внутренние специализированные контуры.

## Roles

### 1. Coordinator / Product
**Owner:** main assistant

**Responsibility:**
- принимать задачу от пользователя
- уточнять рамки задачи при необходимости
- раскладывать работу на подзадачи
- определять, какие роли подключать
- собирать финальный результат
- возвращать итог пользователю

### 2. Quant / DeFi Analyst
**Responsibility:**
- валидировать финансовую логику
- проверять формулы комиссий / доходности / assumptions
- следить за соответствием модели реальному поведению пулов
- выявлять математические риски и неоднозначности

### 3. Data / ETL
**Responsibility:**
- работать с snapshots и источниками данных
- проверять полноту и корректность данных
- документировать pipeline и трансформации
- выявлять проблемы качества данных

### 4. Coder
**Responsibility:**
- реализовывать расчётный движок и продуктовую логику
- писать/править код
- добавлять тесты
- интегрировать data layer с UI/API

### 5. Reviewer / QA
**Responsibility:**
- проверять diff и архитектурные изменения
- ловить edge cases и регрессии
- контролировать согласованность реализации с требованиями
- проверять, что продукт не врёт пользователю

### 6. Ops / Deploy
**Responsibility:**
- окружение
- сборка
- деплой
- эксплуатационные проверки
- логи и стабильность

## Transparency rule

По умолчанию пользователь получает краткую сводку межагентной работы.
По запросу пользователь может запросить детали любого внутреннего этапа или аудит конкретной сессии.
